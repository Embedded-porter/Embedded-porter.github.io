8.烧入固件，是否能启动

  • fdisk：查看当前SD卡挂载的节点，如果是/dev/sdb
  • sudo fdisk -l /dev/sdb：查看是否对应的插入的SD卡大小，
  • ./imxdownload u-boot.bin /dev/sdb：注意当写入速度正常是几百K，最多几M，如果是几百M绝对不正常，系统启动不起来

2. 修改网络驱动

网口PHY芯片的寄存器访问遵循 IEEE 802.3 Clause 22 中定义的 MII 管理接口（MIIM），也就是我们常说的 MDIO/MDC 接口。这个标准明确规定了前几个寄存器的地址和基本功能，使用就有通的驱动使能网口，至少是可以正常使用的

1.查看底板原理图使用的网口IC（使用的mini版，只有网口2）

2. 修改网口PHY地址（目录：mx6ull_tml_emmc.h）

3.修改源码，参考原理图修改对应的IO口

修改之后的源码

  • 官方源码使用了74HC595，去掉相关代码
  • 直接添加复位IO口，添加复位程序

4. 编译新固件，烧入验证

环境变量 描述
ipaddr 开发板 ip 地址，可以不设置，使用 dhcp 命令来从路由器获取 IP 地址。
ethaddr 开发板的 MAC 地址，一定要设置。
gatewayip 网关地址。
netmask 子网掩码。
serverip 服务器 IP 地址，也就是 Ubuntu 主机 IP 地址，用于调试代码

10. Uboot启动linux测试

1. 从EMMC启动

  • 查看当前设备有多少个emmc(SD卡也是一个emmc设备)：mmc list
  • 切换到板载的8G的emmc：mmc dev 1
  • 查看当前emmc的相关信息：mmcinfo
  • 查看emmc分区1的里面的文件：fatls mmc 1:1
  • 将zImage烧写到DDR的0x80800000地址上：fatload mmc 1:1 80800000 zimage
  • 将对应的设备树烧写到DDR的0x83000000地址上：fatload mmc 1:1 83000000 imx6ull-14x14-emmc-4.3-800x480-c.dtb
  • 加载完成，启动设备：bootz 80800000 - 83000000

2.TFTP网络加载

  • 加载kernel镜像：tftp 80800000 zImage
  • 加载设备树：tftp 83000000 imx6ull-14x14-emmc-4.3-800x480-c.dtb
  • 加载完成，启动设备：bootz 80800000 - 83000000

11. 解析bootcmd和bootargs

  • bootcmd：是执行相关指令让u-boot启动内核的指令
  • bootargs：是u-boot传递参数给内核
